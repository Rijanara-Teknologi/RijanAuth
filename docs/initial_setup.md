# RijanAuth Initial Setup Guide

## Overview

RijanAuth automatically seeds the database on first startup, creating essential infrastructure for identity management. This guide covers the initial setup process and security best practices.

## First Run

When RijanAuth starts with an empty database, it automatically:

1. Creates the **master realm** with Keycloak-compatible settings
2. Sets up **authentication flows** (browser, registration, reset credentials, etc.)
3. Creates **default clients** (admin-cli, account-console, broker, etc.)
4. Generates an **admin user** with secure credentials
5. Configures **event logging** for security auditing

## Initial Admin Credentials

On first startup, the console displays:

```
============================================================
  INITIAL ADMIN CREDENTIALS
============================================================
  Username: admin
  Password: <auto-generated-32-char-password>
  Expires:  2026-01-26T12:05:38 (24 hours)
============================================================
  ⚠️  SAVE THESE NOW - They will not be shown again!
============================================================
```

### Important Security Notes

- **Password is shown only once** - save it immediately
- **Password change is required** on first login
- Admin user has `UPDATE_PASSWORD` required action set
- Setup token expires after 24 hours

## Environment Variables

Customize initial setup via environment variables:

| Variable | Default | Description |
|----------|---------|-------------|
| `RIJANAUTH_ADMIN_USER` | `admin` | Initial admin username |
| `RIJANAUTH_ADMIN_EMAIL` | `admin@localhost` | Admin email address |
| `RIJANAUTH_ADMIN_PASSWORD` | (auto-generated) | Initial password (⚠️ use only in controlled environments) |
| `RIJANAUTH_MASTER_REALM_NAME` | `master` | Name of the master realm |
| `RIJANAUTH_SKIP_INITIAL_SEED` | `false` | Skip seeding (for migrations) |

### Example: Custom Admin Setup

```powershell
# Windows PowerShell
$env:RIJANAUTH_ADMIN_USER = "superadmin"
$env:RIJANAUTH_ADMIN_EMAIL = "admin@company.com"
python run.py
```

```bash
# Linux/macOS
export RIJANAUTH_ADMIN_USER="superadmin"
export RIJANAUTH_ADMIN_EMAIL="admin@company.com"
python run.py
```

## First Login Procedure

1. Navigate to `http://localhost:3000/admin/`
2. Log in with the initial credentials shown in console
3. You will be prompted to change your password
4. Set a strong password following your organization's policy
5. Complete any additional required actions

## What Gets Created

### Master Realm Settings

| Setting | Value |
|---------|-------|
| Registration | Disabled |
| Email Verification | Disabled |
| Remember Me | Enabled |
| Login with Email | Enabled |
| Brute Force Protection | Enabled (30 failures max) |
| SSO Session Max | 8 hours |
| Access Token Lifespan | 5 minutes |

### Default Clients

| Client ID | Type | Description |
|-----------|------|-------------|
| `admin-cli` | Public | Command-line administration |
| `account-console` | Public | User self-service |
| `security-admin-console` | Public | Admin web console |
| `broker` | Confidential | Identity brokering |
| `rijanauth-internal` | Bearer-only | Internal service operations |

### Default Client Scopes

- `openid` (default)
- `profile` (default)
- `email` (default)
- `roles` (default)
- `web-origins` (default)
- `address` (optional)
- `phone` (optional)
- `offline_access` (optional)

### Authentication Flows

- **browser** - Standard web login
- **registration** - User self-registration
- **reset-credentials** - Password recovery
- **direct-grant** - Resource Owner Password Credentials
- **clients** - Client authentication
- **first-broker-login** - First-time federated login

## Production Deployment

### Security Checklist

- [ ] Change admin password immediately after first login
- [ ] Configure HTTPS/SSL (set `ssl_required: all` in realm)
- [ ] Set up SMTP for email verification and password resets
- [ ] Review and adjust token lifespans
- [ ] Enable email verification for new users
- [ ] Configure proper CORS origins for clients
- [ ] Set up audit log monitoring
- [ ] Implement backup strategy for the database

### Recommended Production Settings

```python
# In production, set environment variables:
RIJANAUTH_ADMIN_PASSWORD=<strong-password-from-vault>
SECRET_KEY=<secure-random-key>
SQLALCHEMY_DATABASE_URI=<production-database>
```

### Never in Production

- ❌ Use auto-generated passwords visible in logs
- ❌ Skip password change requirement
- ❌ Disable brute force protection
- ❌ Allow registration without email verification
- ❌ Use SQLite database

## Re-running Seeders

Seeders only run when the database is empty. To re-seed:

```bash
# WARNING: This deletes all data!
rm apps/db.sqlite3
python run.py
```

For development, you can also use:

```python
# In Python console
from apps.seeders import run_initial_seed
from apps import db

# Clear database first
db.drop_all()
db.create_all()

# Run seeding
run_initial_seed()
```

## Troubleshooting

### "Could not run database seeding" Error

Check that all model imports are correct and database connection is working.

### Admin User Already Exists

Seeders skip if data already exists. To recreate admin:

```python
from apps.models.user import User
from apps import db

# Find and delete existing admin
admin = User.query.filter_by(username='admin').first()
if admin:
    db.session.delete(admin)
    db.session.commit()
```

### Setup Token Expired

Tokens expire after 24 hours. Reset the admin password directly:

```python
from apps.models.user import User, Credential
from apps.utils.crypto import hash_password
from apps import db

admin = User.query.filter_by(username='admin').first()
cred = admin.get_password_credential()
cred.secret_data = hash_password('new-secure-password')
db.session.commit()
```
